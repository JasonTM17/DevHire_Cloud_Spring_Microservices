# DevHire Cloud v0.4.6 Release Notes

Release target: public credibility, repository governance, and reviewer trust.

## Highlights

- GitHub About, homepage, topics, and branch protection are verified through governance automation.
- Public facade checks now treat the public branch API as the reviewer-facing source of truth when protected branch details require owner permissions.
- Dependabot zero-noise automation can merge clean safe-batch PRs or close/defer pending, failing, unreadable, conflicted, or runtime-major PRs with an audit comment.
- Frontend E2E preview is self-starting for reviewers through `cd frontend && npm run e2e:all`.
- Release evidence includes workflow status checks for CI, Docker Images, Documentation, Security, CodeQL, and E2E Smoke.

## Governance Defaults

- `master` protection requires pull requests, one approval, CODEOWNERS review, conversation resolution, and up-to-date required checks.
- Admin bypass is disabled in the governance payload and settings-as-code baseline.
- Dependabot updates are not blindly merged. Safe updates require readable green checks and clean mergeability.

## Verification

Primary reviewer commands:

```powershell
.\scripts\github-facade-assert.ps1
.\scripts\dependabot-zero-noise.ps1 -DryRun
.\scripts\github-workflow-status.ps1
.\scripts\portfolio-verify.ps1 -Docs -Docker -PublicFacade
cd frontend
npm run e2e:all
```

Runtime Docker verification remains documented in `docs/runtime-evidence-v0.4.md`.

## Compatibility

No business API, database schema, or runtime contract changes are introduced in this release. The release is a public governance, documentation, and evidence hardening pass.
