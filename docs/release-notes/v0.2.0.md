# DevHire Cloud v0.2.0 - Operations-Grade Portfolio Hardening

Status: unreleased

## Theme

v0.2.0 moves DevHire Cloud from a strong microservices demo into an operations-grade production engineering portfolio. The release focuses on evidence that the platform can be reviewed, tested, backed up, observed, and safely deployed through GitOps patterns.

## Added

- Release governance:
  - CODEOWNERS,
  - incident issue template,
  - release checklist,
  - expanded pull request evidence checklist.
- Role-based k6 load suite:
  - public job search,
  - candidate apply,
  - employer review,
  - admin audit,
  - AI assistant fallback.
- Recoverable chaos smoke script:
  - OpenSearch fallback,
  - Kafka outbox pending evidence,
  - AI fallback/circuit evidence,
  - SMTP sandbox outage with internal notification persistence.
- Mailpit local email sandbox:
  - SMTP capture,
  - UI/API on port `8025`,
  - email smoke verification.
- Backup, restore, and DR automation:
  - service-owned PostgreSQL backup,
  - guarded restore,
  - nondestructive DR verification.
- External Secrets and GitOps wiring:
  - Helm `ExternalSecret`,
  - AWS Secrets Manager placeholders,
  - Argo CD sample for External Secrets Operator ordering.
- OpenAPI conformance script for all backend service contracts.
- Recruiter review guide and v0.2 operations case-study documentation.

## Notes

- No AWS resources are applied by CI.
- No secret values are committed.
- Mailpit is the default local email sandbox; Gmail remains optional and secret-backed.
- Reports, backups, generated OpenAPI snapshots, and Terraform state remain ignored.
