# Release Notes v0.3.0

Date: 2026-05-04

## Theme

v0.3.0 professionalizes DevHire Cloud as a senior production/platform engineering portfolio. The release focuses on reviewer experience, version hygiene, dependency maintenance evidence, architecture boundaries, API compatibility, security posture, AWS cloud-readiness evidence, AI assistant safety, frontend product polish, and operations runbooks.

## Highlights

- Added GitHub portfolio facade, owner action checklist, and professional review map.
- Aligned Maven and frontend versions for `0.3.0` development.
- Added Dependabot inventory and curated dependency maintenance policy.
- Added ArchUnit service-boundary tests across services.
- Added API compatibility manifest and runtime OpenAPI compatibility script.
- Added CodeQL and OpenSSF Scorecard workflows plus security evidence documentation.
- Hardened AWS Terraform blueprint documentation, tfvars examples, and cloud-readiness review.
- Added AI prompt-injection guard, AI safety documentation, and eval dataset coverage.
- Polished frontend jobs and assistant experiences with loading, empty, safety, and mobile states.
- Added operations runbooks for alerts, Kafka/outbox, OpenSearch, SMTP, AI provider, and restore drills.

## Compatibility

- Existing public business APIs remain backward compatible.
- API compatibility manifest now lists 39 HTTP endpoints and 6 async contracts.
- No database ownership boundary changed.
- No Terraform apply or AWS credential is required.
