# Release Notes v0.4.0

Date: unreleased

Status: in development on `master`.

## Theme

v0.4.0 focuses on runtime proof, release hygiene, GitHub publication polish, dependency triage, and reviewer-friendly evidence. It keeps the business API stable and improves how reviewers verify that the platform runs, degrades, and is maintained like a production portfolio.

## Planned Highlights

- Reconciled v0.3 public release evidence and moved `master` to `0.4.0-SNAPSHOT`.
- GitHub About/Homepage/Topics checklist and optional owner-token publication flow.
- Curated Dependabot triage evidence for safe batches and deferred major upgrades.
- Docker runtime preflight checks before full-stack smoke verification.
- Sanitized v0.4 runtime evidence pack that summarizes generated local reports without committing raw tokens or payloads.

## Compatibility

- Existing public business APIs remain backward compatible.
- AWS Terraform remains blueprint-only; no `terraform apply` is required.
- Runtime reports remain ignored under `reports/`.

## Verification Target

- `mvn -T1 clean verify`
- `cd frontend && npm run typecheck && npm run build`
- `.\scripts\portfolio-verify.ps1 -Docs -Docker`
- `.\scripts\portfolio-verify.ps1 -Runtime -GatewayUrl http://localhost:8080` when Docker Desktop is available
