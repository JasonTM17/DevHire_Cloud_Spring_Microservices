# DevHire Cloud v0.4.6 Public Credibility Evidence

Date: 2026-05-05  
Release type: public facade, E2E trust, and reviewer evidence hardening  
Branch: `master`

## Summary

v0.4.6 focuses on the parts a senior reviewer sees immediately: GitHub facade status, branch protection proof, Dependabot noise, self-starting browser smoke, and a tighter README first viewport. No business API, database schema, or secret-bearing configuration was changed.

Owner-authenticated GitHub API has applied and verified the public facade. Tagging should still wait until the latest post-apply documentation commit has green GitHub Actions.

## What Changed

| Area | Evidence |
|---|---|
| GitHub facade apply workflow | `Repository Governance` now supports `verify-only`; apply modes run hard assertions after mutation |
| Branch protection context audit | `github-check-contexts.ps1` supports strict `-RequireAvailable` for owner workflow and local soft-unavailable reporting without a token |
| Public facade assertion | `github-facade-assert.ps1` fails hard after owner apply if About/Homepage/Topics or `master protected=true` are missing |
| Dependabot burn-down | Manual `Dependabot Curation` workflow can label safe batches and close deferred major PRs without auto-merge |
| Frontend E2E trust | `cd frontend && npm run e2e:all` builds, prepares the Next.js standalone preview, waits, runs desktop + mobile Playwright smoke, and stops the server |
| Public evidence gate | `public-portfolio-audit.ps1` summarizes facade, repository health, Dependabot categories, screenshot manifest, docs quality, and optional E2E |
| README first viewport | Public state, quick links, and reviewer commands are visible before long narrative |
