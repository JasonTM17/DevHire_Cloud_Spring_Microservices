## Summary

v0.4.6 focuses on the parts a senior reviewer sees immediately: GitHub facade status, branch protection proof, Dependabot noise, self-starting browser smoke, and a tighter README first viewport. No business API, database schema, or secret-bearing configuration was changed.

Owner-authenticated GitHub API has applied and verified the public facade. v0.4.7 hardening completed the final tag gate: public-limited branch protection reads no longer fail falsely, Dependabot zero-noise cleanup reduced the open PR count to 0, admin bypass is disabled in the governance payload, and workflow status is green for the latest `master` head SHA.

Zero-noise policy:

- safe PRs merge only when `mergeable_state=clean` and readable checks are green;
- safe PRs with pending, failing, conflicted, or unreadable checks close with a scheduled dependency refresh comment;
- runtime-major/manual-review PRs close with a migration rationale;
- no PR is auto-merged when check status cannot be read.
