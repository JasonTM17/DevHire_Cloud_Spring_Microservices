# DevHire Cloud v1.0.0 Release Evidence

## Summary

This file is the canonical v1.0.0 evidence checklist. It should be updated at release time with the final commit SHA, workflow links, and command results. Until then, it records the exact acceptance gates required for v1.0.0.

## Required State Before Tag

| Evidence | Required state |
|---|---|
| Branch governance | `master` protected, no direct push, release PR merged through review |
| GitHub facade | About, homepage, topics, releases, and branch protection applied |
| Dependabot | 0 open PRs or fully documented curated queue |
| CI | GitHub `CI` workflow green on `master` |
| Docker | Docker image matrix green |
| Docs | Documentation workflow green |
| Security | Security workflow green |
| CodeQL | CodeQL workflow green |
| E2E | E2E Smoke workflow green |
| Terraform | Terraform workflow green |
| Backend | `mvn -T1 clean verify` passes |
| Frontend | `npm run typecheck`, `npm run build`, and `npm run e2e:all` pass |
| Cloud | `v1-cloud-evidence.ps1` passes without AWS credentials |
| Runtime | `v1-runtime-evidence.ps1` passes when Docker stack is available |
| Docs parity | Vietnamese, English, and Japanese docs pass parity checks |
| Evidence manifest | All required docs, scripts, screenshots, and runbooks exist |

## Reviewer Commands

Fast release evidence:

```powershell
.\scripts\v1-release-verify.ps1 -Cloud
```

Backend and frontend evidence:

```powershell
.\scripts\v1-release-verify.ps1 -Backend -Frontend -Cloud
```

Runtime evidence:

```powershell
docker compose up -d --build
.\scripts\v1-runtime-evidence.ps1 -GatewayUrl http://localhost:8080
```

Cloud evidence:

```powershell
.\scripts\v1-cloud-evidence.ps1
```

Demo data evidence:

```powershell
.\scripts\v1-demo-data-verify.ps1
.\scripts\v1-demo-data-verify.ps1 -FromDocker
```

## Current Release Notes

- Current public release remains `v0.4.6`.
- Current hardening branch contains v0.5.1 evidence and awaits protected-branch approval before merge.
- v1.0.0 must not be tagged until all v1 acceptance gates pass on `master`.

## Non-Claims

- No real AWS deployment is claimed without a credentialed apply phase.
- No live customer traffic is claimed.
- No external penetration test is claimed unless a future report is attached.
- Generated reports under `reports/` remain local and ignored.
- Secrets, runtime tokens, tfstate, kubeconfig, backups, and `.env` files remain untracked.
