# DevHire Cloud v0.4.8 Cloud Blueprint Evidence

v0.4.8 focuses on cloud blueprint credibility and reviewer verification. It does not change business APIs, database schemas, or runtime secrets. It makes the AWS/Kubernetes deployment path easier to review without requiring an AWS account.

## Scope

| Area | Result |
|---|---|
| Raw Kubernetes | Uses `sha-REPLACE_WITH_GIT_SHA` placeholders instead of `latest` and now includes `ai-service` |
| Helm defaults | Default chart values are safe to render, do not use `latest`, and do not create example secrets by default |
| Helm AWS values | AWS staging/prod values use `imagePullPolicy: Always` with immutable tag posture |
| GitOps | Local Argo CD sample tracks `master`, matching the repository default branch |
| Terraform | AWS blueprint remains validate-only and safe; no `terraform apply` is run |
| Reviewer command | `scripts/cloud-verify.ps1` validates Terraform, Helm, Kustomize, kubeconform, and cloud policy assertions |
| Local cleanup | `scripts/clean-local-artifacts.ps1` also handles Terraform cache and generated frontend/runtime artifacts |
| Coverage posture | Low-module thresholds are ratcheted for `ai-service`, `api-gateway`, `auth-service`, and `common-lib` |

## Commands

These commands are the intended v0.4.8 acceptance path:

```powershell
.\scripts\cloud-verify.ps1
.\scripts\portfolio-verify.ps1 -Docs -Docker -Cloud
.\scripts\terraform-validate.ps1
docker compose config --quiet
kubectl kustomize deploy\k8s
.\scripts\docs-quality.ps1
.\scripts\evidence-manifest-verify.ps1
.\scripts\repo-hygiene.ps1
.\scripts\domain-placeholder-audit.ps1
.\scripts\professionalism-audit.ps1
mvn -T1 clean verify
.\scripts\check-coverage.ps1
```

Frontend verification is unchanged:

```powershell
cd frontend
npm run typecheck
npm run build
npm run e2e:all
```

## Cloud Verification Contract

`cloud-verify.ps1` is intentionally Docker-first:

- if local `helm` is missing, it uses `alpine/helm`;
- if local `kubectl` is missing, it uses `bitnami/kubectl`;
- kubeconform runs through a container image;
- Terraform validation uses the existing safe script and does not apply infrastructure;
- reports are written under ignored `reports/cloud-verify/`.

This means reviewers can validate the cloud blueprint on a workstation that has Docker but does not have the full cloud CLI stack installed.

## Non-Goals

- No AWS deployment was performed.
- No AWS credentials were used or required.
- No runtime secret was committed.
- No production domain was claimed; placeholder domains remain explicit replacement markers.
- No business endpoint changed.
