# DevHire Cloud v0.6.0 Product And Code Assessment Evidence

## Summary

`v0.6.0` is the public release that makes DevHire Cloud reviewer-ready as a product surface, not only as a microservices codebase. It consolidates the v0.6 Stitch/UI work, Java code-assessment runner boundary, employer/admin review surfaces, runtime monitoring, branch cleanup, and Docker Hub image verification path.

## Shipped Scope

- Candidate marketplace UI: ITViec-inspired search-first jobs page, job detail, salary/location/company emphasis, clean mobile states, and no debug/offline copy on primary screens.
- Candidate Code Studio: `Cloud Architecture Challenge`, `CandidateSolution.java`, visible cases, custom input, stdout/stderr/compile output, timeout/runner-unavailable states, submission history, locked/submitted states, and hidden payload redaction.
- Employer review: assignment endpoint, duplicate-safe assignment, dossier evidence, runtime/static score split, visible/hidden aggregate, risk flags, sanitized output, immutable raw score, and `PASS`/`HOLD`/`REJECT` actions.
- Admin/ops: challenge authoring validation, assessment summary, runner health, fail-closed metadata, queue/latency metrics, Grafana/Prometheus coverage, and release/branch evidence.
- Runtime: deterministic local runner, Judge0-compatible adapter contract, Java harness, normalized stdout comparison, policy blocking, verdict evidence, and release smokes.
- Release hygiene: no open PRs, no unmerged local/remote branches, zero Dependabot queue, restored branch protection, green master workflows, and Docker Hub image verification.

## Docker Hub Verification

Docker Hub mirrors use:

```text
docker.io/nguyenson1710/devhire-cloud-<service>:v0.6.0
```

The full set is checked with:

```powershell
.\scripts\dockerhub-image-verify.ps1 -Tags v0.6.0
```

Smoke against pulled images uses the Docker Hub compose overlay:

```powershell
$env:IMAGE_TAG = "v0.6.0"
$env:DOCKERHUB_NAMESPACE = "nguyenson1710"
docker compose -f docker-compose.yml -f deploy/docker-compose.dockerhub.yml up -d --pull always --no-build
.\scripts\code-assessment-smoke.ps1 -GatewayUrl http://localhost:8080
.\scripts\runtime-observability-smoke.ps1 -GatewayUrl http://localhost:8080
```

## Evidence Rules

- Generated runtime reports stay ignored under `reports/`.
- Hidden code-assessment cases are not exposed to candidate APIs, UI, screenshots, or release docs.
- Deterministic runner mode is acceptable for local/reviewer Docker smoke only.
- Production runner deployments must use Judge0 mode and fail closed when `JUDGE0_BASE_URL` is missing.
- AWS remains an apply-ready blueprint until a separate credentialed deployment phase is approved.
