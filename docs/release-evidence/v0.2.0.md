- recoverable chaos and resilience scenarios,
- Mailpit SMTP sandbox with optional Gmail SMTP mode,
- backup, restore, and disaster recovery runbooks,
- External Secrets and GitOps wiring,
- OpenAPI conformance checks,
- operations screenshots captured from the real local stack,
- unified local verification runner.

## Honest Limitations

- AWS Terraform is a blueprint and is not applied automatically.
- Gmail SMTP is optional and must be configured through local `.env` or a secret store; local Docker uses Mailpit by default.
- Generated reports, backups, Terraform state, kubeconfigs, and secret material are intentionally ignored.
- GitHub About, topics, branch protection, and release tag creation remain manual repository-owner actions.

## Manual GitHub Release Steps

1. Fill the repository About and topics from `docs/github-profile.md`.
2. Enable branch protection for `master`.
3. Keep future release tags on commits that have passed CI, Docker, security, and documentation workflows.
4. Use the release workflow to create the GitHub Release and publish GHCR images from tags.
